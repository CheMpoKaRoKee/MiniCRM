# Test log

## Проверенный сценарий

Собран и проверен рабочий прототип мини-CRM для юридической команды.

### Основной поток

1. Пользователь открывает Web App.
2. Добавляет клиента и контактные данные.
3. Выбирает статус и предпочтительный способ связи.
4. Данные сохраняются в Google Sheets.
5. Интерфейс обновляет список клиентов и счётчики.
6. Email-уведомление отправляется ответственному.
7. Результат операции записывается в журнал.

## Проверенные функции

- создание базовой структуры листов;
- добавление клиента;
- обязательная валидация имени и основного телефона;
- проверка допустимого статуса;
- проверка способа связи;
- чтение клиентов из Google Sheets;
- изменение статуса и комментария;
- подсчёт клиентов по статусам;
- нормализация значений из таблицы перед отправкой во frontend;
- email-уведомление при добавлении клиента;
- повторная отправка уведомления;
- запись операций в `Logs`;
- чтение публичных настроек из `Settings`.

## Контрольная диагностика

После запуска `setupPrototypeSheets` и `debugBootstrapData` ожидаются:

```text
parsedClientsCount > 0
statusesCount = 13
contactMethodsCount = 4
```

## Что проект демонстрирует

- перевод бизнес-требований в работающий MVP;
- использование Google Sheets как прозрачного хранилища;
- backend-логику и Web App на Google Apps Script;
- валидацию и нормализацию данных;
- журналирование действий;
- автоматизацию email-уведомлений.

## Моя зона ответственности

Самостоятельно определил сценарий CRM, структуру таблиц, набор юридических статусов, UX-логику, стек и требования к уведомлениям.

AI использовался как инструмент ускорения разработки: для чернового кода, отладки Apps Script, проверки UX, обработки ошибок и подготовки документации.

## Ограничение отчёта

Точное время выполнения тестового задания отдельно не фиксировалось, поэтому в отчёте не указывается приблизительная оценка.
